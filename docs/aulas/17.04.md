
# Configurando o Ambiente de Desenvolvimento

## 1. Preparação do Ambiente de Desenvolvimento

Para garantir que todas as ferramentas funcionem corretamente, seguiremos uma estrutura de pastas organizada diretamente na raiz do seu sistema (Partição C). Certifique-se de que as ferramentas básicas estão descompactadas na partição C do seu computador. Caso não, devemos baixar o arquivo **Ambiente Git / Maven** do 
Materiais de Aula e descompactá-lo na partição C do seu computador.   

Vamos entender a estrutura de diretórios proposto. Ao descompactar para a partição C, você encontrar as seguintes pastas: 

1. `C:\dev` pasta que irá centralizar todos os artefatos e ferramentas do que vamos realizar neste curso.
2. `C:\dev\tools` pasto que estará as ferramentas essenciais: o **Git Portable** e o **Apache Maven**.

> **Obs1:** Manter essa estrutura de pastas/caminho é fundamentoa para os próximos passos.  
> **Obs2:** Esqueçam os arquivos `Iniciar_Ambiente.bat` que se encontra no diretório `tools`

## 2. Configuração do Maven no VS Code

Agora, vamos fazer o VS Code "enxergar" e integrar o ambiente de desenvolvimento com as ferramentas que preparamos no diretório `C:\dev\tools`. 
O primeiro passo é garantir que o editor suporte a linguagem Java e o gerenciador de dependências **Maven**.  

### Instalação da Extensão
Para que o VS Code entenda o código Java, precisamos de um pacote de extensões oficial:
1. No menu lateral esquerdo do VS Code, clique no ícone de Extensions (ou pressione `Ctrl + Shift + X`).
2. Na barra de busca do _Marketplace_, digite: "Extension Pack for Java".
3. Verifique se o autor é a _Microsoft_. Caso ainda não esteja instalado, clique no botão azul **Install**.

> **Obs1:** Esse pacote já instala automaticamente o suporte ao Maven e ao depurador Java.

<img width="1179" height="269" alt="image" src="https://github.com/user-attachments/assets/24097a16-d0d8-4a32-8261-678bd892fa86" />

### Configuração do Executável 
Para garantir que o VS Code use exatamente o Maven que colocamos na pasta `C:\dev\tools`:

1. Pressione `Ctrl + ,` (Vírgula) para abrir as Settings (Configurações).
2. Pesquise por `maven.executable.path`.
3. No campo de texto, informe o caminho completo até o arquivo mvn dentro da sua pasta de ferramentas, por exemplo: `C:\dev\tools\apache-maven-3.9.14\bin\mvn.cmd`

<img width="1184" height="396" alt="image" src="https://github.com/user-attachments/assets/f60273c8-b521-461c-8a3c-ff01097fa167" />

### Configuração do Terminal (Maven)
Para garantir que o VS Code reconheça os comandos do Maven (como `mvn clean install`) diretamente no terminal integrado, 
precisamos "ensinar" ao ambiente onde as ferramentas estão localizadas.

1. No VS Code, pressione `Ctrl + Shift + P` para abrir a **Paleta de Comandos**.
2. Digite `Open User Settings (JSON)` e selecione essa opção.
3. Dentro das chaves `{ }` do arquivo, adicione o seguinte bloco de código (certifique-se de que o caminho corresponde à versão do Maven que você tem na pasta tools):

``` JSON
    "terminal.integrated.env.windows": {
        "PATH": "C:\\dev\\tools\\apache-maven-3.9.14\\bin;${env:PATH}",
        "MAVEN_HOME": "C:\\dev\\tools\\apache-maven-3.9.14"
    }
```
4. Reinicie o VS Code, abra qualquer terminal integrado e digite `mvn -version`

## 3. Configuração do Git no VS Code
Agora vamos integrar o Git Portable ao VS Code. Isso permitirá que você controle as versões do seu código, faça commits e sincronize com o GitHub diretamente pela interface do editor.

### Localizando o Executável do Git

Para que o VS Code saiba onde o Git foi "instalado" (descompactado):

1. Pressione `Ctrl + ,` (Vírgula) para abrir as Settings.
2. Pesquise por `git.path`.
3. Clique no link "Edit in settings.json".
 <img width="1185" height="388" alt="image" src="https://github.com/user-attachments/assets/d1d81b9d-1fe2-42f2-b457-28f33c2135dd" />
  
5. No arquivo que abrir, adicione (ou edite) a linha apontando para o executável dentro da sua pasta `C:\dev\tools`: `"git.path": "C:\\dev\\tools\\Git\\bin\\git.exe"`  
 <img width="1188" height="286" alt="image" src="https://github.com/user-attachments/assets/a034d444-8eea-418f-b2a5-b7c1294212f5" />

### Configurando o Git Bash como Terminal 
O Git Bash oferece comandos Linux que facilitam a vida. Vamos configurá-lo como o terminal de preferência do seu VS Code.   

1. Na mesma tela de Settings (JSON) que você abriu no passo anterior, adicione as seguintes configurações para registrar o perfil do Git Bash:
   ``` JSON 
        "terminal.integrated.profiles.windows": {
            "Git Bash Portable": {
                "path": "C:\\dev\\tools\\PortableGit\\bin\\bash.exe",
                "icon": "terminal-bash"
            }
        },
        "terminal.integrated.defaultProfile.windows": "GitBash"
   ```
<img width="1189" height="439" alt="image" src="https://github.com/user-attachments/assets/4f397a13-b2e1-4ec1-a88b-35cd263712f0" />

2. Salve o arquivo (`Ctrl + S`).

### Configuração de Identidade (Global)
Para que o Git não reclame na hora de salvar seu trabalho, você precisa se identificar no terminal (faça isso apenas uma vez):

1. Após reiniciar o VS Code, abra um novo terminal no VS Code (`Ctrl + '`). Note que agora ele deve abrir como bash.
2. Digite os comandos abaixo substituindo pelos seus dados:
 - `git config --global user.name "Seu Nome Completo"`
 - `git config --global user.email "seu.email@exemplo.com"`
